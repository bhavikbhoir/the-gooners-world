# The Gooners World

A full-stack Arsenal FC fan site with live match data, AI-powered match analysis, a conversational AI assistant, and an admin panel for publishing branded match result posts to Instagram and X — built as a portfolio flagship project.

**Live:** https://the-gooners-world.web.app

## Features

### Home
- **Live score banner** — polls the backend every 30 s during active matches; derives stoppage-time minute labels (45+, 90+) from kickoff time when the upstream API omits the field
- **Match countdown** — live countdown timer to Arsenal's next fixture
- **AI match prediction** — pre-match preview generated by Claude Sonnet 4.6 via AWS Bedrock, conditioned on the upcoming fixture and Arsenal's last five results; cached 1 hour per fixture pair; shareable via X, WhatsApp, and copy-to-clipboard
- **Fixtures & results** — last 2 results and next 2 upcoming matches, each finished match annotated with an AI-generated post-match summary (cached 24 hours)
- **Scrolling news ticker** — top Arsenal headlines from NewsData.io, deduplicated and cached 30 minutes
- **Latest news cards** — top 5 articles with social share buttons; articles with no image use full card width with a red accent border rather than a grey placeholder

### Stats & Fixtures
- **Full fixtures list** — all Arsenal matches across all competitions with venue and referee
- **Premier League table** — full 20-team standings with Arsenal row highlighted; form guide computed for all 20 teams from match results (not the upstream API field which is unavailable on the free tier)
- **PL top scorers** — top 20 scorers with goals, assists, and penalties
- **Champions League journey** — all UCL matches grouped by stage (League Stage → R16 → QF → SF), with crests and scorelines

### News Center
- **Full news feed** — up to 20 Arsenal articles from NewsData.io, filtered and deduplicated
- **Smart filtering** — semantic deduplication (60% keyword overlap) collapses same-story duplicates from multiple outlets; broad regex catches irrelevant titles (celebrity gossip, lifestyle, non-Arsenal match scores); AI curation (Claude Haiku) in the backend provides the primary quality layer
- **Keyword search and source filter** — client-side filtering with no additional API calls
- **Share buttons** — X, WhatsApp, copy link on every article

### Arsenal AI Assistant
- **Floating chat widget** — persistent across all pages, powered by AWS Bedrock Agent Core
- **Session-based conversation** — unique session ID per page load; 30-message limit with graceful cutoff message
- **Structured response rendering** — numbered list responses automatically formatted as table rows

### Squad
- **Full squad browser** — Arsenal first-team players grouped by position (Goalkeepers, Defenders, Midfielders, Forwards) with nationality and age

### Admin Panel (`/admin`)
- **Lambda-authenticated access** — password verified server-side against SSM; never stored in the client bundle; returns a signed 8-hour HMAC-SHA256 Bearer token; session persists across page refreshes (sessionStorage) but clears on tab close
- **Match picker** — select any finished Arsenal match from a dropdown; match detail (goals, scorers, referee, venue) fetched automatically from football-data.org
- **Canvas image generation** — one click generates a 1080×1080 Instagram-ready image entirely in the browser using the HTML Canvas API; no server dependency or binary packages required
  - **PL theme** — dark charcoal background, red accent bar, red score, red footer band
  - **UCL theme** — deep navy background, gold accent bar, gold score, UEFA-navy footer with gold stripe; auto-detected from the competition name
  - **Photo background mode** — upload any photo first; Generate overlays the match data on top with a dark scrim, stronger gradient at the header/footer zones, accent-colour tint in the score area, and text drop-shadows for legibility on any photo
- **Lightbox preview** — Preview button opens the full 1080×1080 image in a modal overlay on the page (Escape or click outside to close); Download button saves the PNG locally
- **Claude-generated post copy** — one click calls Claude Sonnet 4.6 to write a branded Instagram caption and X post from the match result, scorers, and recent form
- **Editable drafts** — review and edit both captions before posting; X tab shows live character counter
- **Publish to Instagram** — uploads image to private S3 (5-min pre-signed URL), creates media container, publishes via Instagram Graph API
- **Publish to X** — posts via OAuth 1.0a HMAC-SHA1 signed request to X API v2 (requires X Basic plan)
- **Post to both** — single click publishes to both platforms simultaneously

## Architecture

```
Browser (React 18 SPA)
│   Firebase Hosting — CDN, SPA rewrite, CSP headers, 1yr asset cache
│
├── football-data.org v4 API
│       GET /proxy/football?type=matches|standings|pl-results|scorers|cl-matches|live|squad|match
│       └── Lambda (proxy/football) → football-data.org
│
├── NewsData.io API
│       GET /proxy/news
│       └── Lambda (proxy/news) → newsdata.io → semantic dedup → Claude Haiku curation
│
├── AWS Bedrock — AI text generation
│       GET /proxy/ai?type=prediction|summary&data={...}
│       └── Lambda (proxy/ai) → Bedrock Claude Sonnet 4.6
│
├── AWS Bedrock Agent Core — conversational AI
│       POST /agent/chat  { message, sessionId }
│       └── Lambda (agent/chat) → Bedrock Agent runtime → agentHandler (9 tools)
│
└── Admin
        POST /admin/auth     { password }        → adminAuth Lambda → SSM comparison → Bearer token
        POST /admin/generate { matchData }       → generateMatchPost Lambda → Claude Sonnet 4.6
        POST /admin/publish  { platform, image } → publishPost Lambda → S3 → Instagram / X API

All routes through API Gateway (x-api-key auth + rate limiting: 1,000 req/day, 10 req/sec)
API keys and credentials stored in AWS SSM Parameter Store — never in the client bundle
```

## Key Technical Highlights

**Lambda-based admin authentication**
The admin password is verified server-side in `adminAuth` Lambda against an SSM SecureString. On success it returns a stateless HMAC-SHA256 token (`{timestamp}.{signature}`) signed with a separate SSM secret. Downstream Lambdas verify the token signature and expiry independently — no shared session store needed. The token lives in `sessionStorage` so it survives page refreshes but is discarded when the tab is closed.

**Client-side canvas image generation**
Match result images are generated entirely in the browser using the HTML Canvas API — no Lambda, no binary packages, no image processing library. The canvas draws the background (dark theme or user-uploaded photo), overlays gradients and colour-tinted glows, then renders team names (auto-scaling font from 96px down to fit any name length), a dominant score at up to 240px, and goal scorer rows in a two-column layout. Two themes are applied automatically: PL (red/black) and UCL (gold/navy) based on the competition name.

**Photo background with scrim compositing**
When the admin uploads a background photo, the canvas draws it cover-cropped to 1080×1080, applies a uniform dark scrim (`rgba(0,0,0,0.52)`), stronger linear gradient darkening at the header and footer zones, and a subtle radial accent-colour tint in the score centre. All text is rendered with canvas `shadowBlur` drop shadows so it remains legible on any photo content.

**Private S3 with pre-signed URLs**
Instagram requires a public image URL, but the S3 bucket stays fully private. The `publishPost` Lambda generates a 5-minute GetObject pre-signed URL immediately after upload. Instagram fetches the image in seconds; the URL expires and becomes unreachable automatically.

**Form guide for all 20 PL teams**
The Premier League standings API's `form` field returns null on the free tier. Instead, `fetchPLResults()` fetches all finished PL matches separately, builds a per-team result history from match scores, and computes last-5 form strings for every team in the table independently.

**Client-side caching via localStorage**
Every data type has a dedicated TTL enforced before any network call — from 30 s for live scores to 24 hours for squad data. AI responses are keyed by fixture pair (`home_away`) so the same prediction is never regenerated within its TTL window. Cache keys are versioned so stale data is flushed automatically when the format changes.

**Semantic news deduplication**
Articles are fingerprinted by extracting significant words (stop words removed, ≥3 chars). If two articles share ≥60% of their keyword sets, only the first is kept. This runs in the backend Lambda before AI curation and again in the frontend as a safety net.

**Rate-limit-aware live polling**
`LiveScore` adapts its polling interval dynamically: 30 s when a match is live, 60 s at all other times, and polls stop immediately on component unmount via a cancelled-flag pattern — preventing both memory leaks and unnecessary API calls.

**Security posture**
CSP, `X-Frame-Options: DENY`, `X-XSS-Protection`, and `Referrer-Policy` headers set at the CDN layer in `firebase.json`. Origin validation in every Lambda. API Gateway enforces key auth and rate limits. No secrets touch the client bundle.

## Setup & Run

```bash
# 1. Install dependencies
npm install

# 2. Create .env.local with your keys
VITE_API_BASE=https://<id>.execute-api.us-east-1.amazonaws.com/dev
VITE_API_GW_KEY=<api-gateway-key>
VITE_FOOTBALL_API_KEY=<football-data.org-key>   # dev proxy only
VITE_NEWS_API_KEY=<newsdata.io-key>             # dev proxy only

# 3. Start dev server (Vite proxy handles CORS and key injection)
npm start

# 4. Production build + deploy
npm run build
firebase deploy
```

The admin panel at `/admin` authenticates against the deployed Lambda — no local env vars needed for the password.
