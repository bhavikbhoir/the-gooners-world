## Features

### Home
- **Live score banner** — polls the backend every 30 s during active matches; derives stoppage-time minute labels (45+, 90+) from kickoff time when the upstream API omits the field
- **Match countdown** — live countdown timer to Arsenal's next fixture
- **AI match prediction** — pre-match preview generated by AWS Bedrock (Amazon Nova Micro), conditioned on the upcoming fixture and Arsenal's last five results; cached 1 hour per fixture pair
- **Fixtures & results** — last 2 results and next 2 upcoming matches, each finished match annotated with an AI-generated post-match summary (cached 24 hours)
- **Scrolling news ticker** — top Arsenal headlines from NewsData.io, deduplicated and cached 30 minutes
- **Latest news cards** — top 5 articles with image fallbacks and social share buttons (Twitter/X, WhatsApp, copy link)

### Match Center
- **Full fixtures list** — all Arsenal matches across all competitions with venue and referee
- **Premier League table** — full 20-team standings with Arsenal row highlighted
- **PL top scorers** — top 20 scorers with goals, assists, and penalties
- **Champions League journey** — all UCL matches grouped by stage (League Stage → R16 → QF → SF), with crests and scorelines

### News Center
- **Full news feed** — up to 20 Arsenal articles from NewsData.io, deduplicated by title
- **Keyword search and source filter** — client-side filtering with no additional API calls
- **Share buttons** — Twitter/X, WhatsApp, copy link on every article

### Arsenal AI Assistant
- **Floating chat widget** — persistent across all pages, powered by AWS Bedrock Agent Core
- **Session-based conversation** — unique session ID per page load; 30-message limit with graceful cutoff message
- **Structured response rendering** — numbered list responses automatically formatted as table rows

### Squad
- **Full squad browser** — Arsenal first-team players grouped by position (Goalkeepers, Defenders, Midfielders, Forwards) with nationality and age

## Architecture

```
Browser (React 18 SPA)
│   Firebase Hosting — CDN, SPA rewrite, CSP headers, 1yr asset cache
│
├── football-data.org v4 API
│       GET /proxy/football?type=matches|standings|scorers|cl-matches|live|squad
│       └── Lambda (proxy/football) → football-data.org → response + Cache-Control
│
├── NewsData.io API
│       GET /proxy/news
│       └── Lambda (proxy/news) → newsdata.io → deduplicated articles
│
├── AWS Bedrock (Amazon Nova Micro) — AI text generation
│       GET /proxy/ai?type=prediction|summary&data={...}
│       └── Lambda (proxy/ai) → Bedrock InvokeModel
│
└── AWS Bedrock Agent Core — conversational AI
        POST /agent/chat  { message, sessionId }
        └── Lambda (agent/chat) → Bedrock Agent runtime

All routes through API Gateway (API key auth + rate limiting: 500 req/day, 5 req/sec)
API keys stored in AWS SSM Parameter Store — never in the client bundle
```

## Key Technical Highlights

**Client-side caching via localStorage**
Every data type has a dedicated TTL enforced before any network call is made — from 30 s for live scores (bypassed entirely with `cache: 'no-store'`) to 24 hours for squad data. AI responses are keyed by fixture pair (`home_away`) so the same prediction is never regenerated within its TTL window. This keeps the site fast and minimises API Gateway costs.

**Rate-limit-aware live polling**
`LiveScore` adapts its polling interval dynamically: 30 s when a match is live, 60 s at all other times, and polls stop immediately on component unmount via a cancelled-flag pattern — preventing both memory leaks and unnecessary API calls.

**Dual-mode API routing**
A single `isDev` flag switches all four API modules between Vite dev-server proxy (keys injected server-side, CORS handled automatically) and direct production fetch with `x-api-key`. No conditional build flags or separate entry points required.

**AI integration with context awareness**
Match predictions are generated with Arsenal's last five results as context, formatted as structured text and passed to Bedrock alongside fixture metadata. Post-match summaries include competition, stage, and scoreline. Both are cached aggressively so Bedrock is only called once per unique fixture.

**Security posture**
CSP, `X-Frame-Options: DENY`, `X-XSS-Protection`, and `Referrer-Policy` headers set at the CDN layer in `firebase.json`. Origin validation in Lambda. API Gateway enforces key auth and rate limits. No secrets touch the client bundle.

**Code splitting**
All five pages are `React.lazy`-loaded. The `ChatWidget` is eagerly mounted so the floating button renders immediately without waiting for page routes.

## Setup & Run

```bash
# 1. Install dependencies
npm install

# 2. Create .env.local with your keys
VITE_API_BASE=https://<id>.execute-api.us-east-1.amazonaws.com/dev
VITE_API_GW_KEY=<api-gateway-key>
VITE_FOOTBALL_API_KEY=<football-data.org-key>
VITE_NEWS_API_KEY=<newsdata.io-key>

# 3. Start dev server (Vite proxy handles CORS and key injection)
npm start

# 4. Production build + deploy
npm run build
firebase deploy
```
